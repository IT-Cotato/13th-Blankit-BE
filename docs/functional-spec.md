# Blankit 기능 명세서

> 백엔드 API 설계 기준 문서. 각 항목의 `[P1]`/`[P2]`/`[P3]`은 우선순위 1·2·3순위를 의미한다.
> 우선순위/예상시간/추천 관련 계산 로직은 `docs/logic-spec.md` 참고.

## 2. 홈 화면

### 2.1 모든 과업 카드 `[P1]`
1. 모든 과업 카드는 과업명, 사용자가 피드백 화면에서 입력한 과업 메모, 진행바를 표시한다.
2. 카테고리별로 사용자가 지정한 색상과 아이콘을 적용한다.
3. 아직 과업 메모를 입력하지 않은 경우 해당 영역은 표시하지 않고 공백 상태로 유지한다.
4. 진행바 중앙에는 전체 진행률을 10등분한 구간을 기준으로 현재 진행 구간에 해당하는 숫자(1~10)를 표시한다.
5. 과업 완료 시 진행바 중앙의 숫자는 체크 아이콘으로 변경되며 약 3초간 표시된다.
6. 이후 "과업을 완료했습니다." 토스트 메시지를 표시하고 해당 과업은 홈 화면에서 제거되어 마이페이지의 완료한 과업 목록으로 이동한다.
7. 진행바 색상은 우선순위에 따라 상(빨간색), 중(노란색), 하(초록색)으로 구분하여 표시한다.

### 2.2 오늘의 권장 시간 `[P1]`
1. 오늘의 권장 시간을 표시한다.
2. 권장 시간 로직은 logic-spec의 "5. 오늘 최소 시간 로직" 참고.

#### 2.2.1 과업 시작 버튼 클릭 시 `[P2]`
1. 사용자는 과업을 플레이리스트에 추가한 후 플레이리스트 바텀시트를 통해 재생 버튼을 클릭하여 과업을 시작할 수 있다.
2. 사용자는 플레이리스트에 과업을 추가한 상태에서 홈 화면의 과업 시작 버튼을 클릭하여 현재 플레이리스트 순서를 유지한 채 플레이리스트의 첫 번째 과업 플레이 화면으로 이동한다.
3. 사용자가 플레이리스트에 과업을 추가하지 않은 상태에서 과업 시작 버튼을 클릭한 경우, 시스템은 우선순위가 가장 높은 과업부터 최대 3개의 과업을 자동으로 플레이리스트에 추가한 후 첫 번째 과업 플레이 화면으로 이동한다.

### 2.3 우선순위 과목 추천

#### 2.3.1 우선순위 추천 로직 `[P2]`
1. 사용자가 수행해야 할 과업을 우선순위 상·중·하로 정렬하여 추천한다.
2. 우선순위 정렬 로직은 logic-spec의 "1. 우선순위 추천 로직" 참고.

#### 2.3.2 추천 개수 `[P2]`
1. 우선순위에 따라 최대 3개의 과업을 추천한다.
2. 과업은 우선순위 순으로 정렬한 후 상위 3개의 과업만 사용자에게 제공한다.
3. 추천된 과업 중 하나를 사용자가 완료한 경우, 다음 우선순위의 과업을 추천 목록에 추가하여 최대 3개의 추천 과업 수를 유지한다.

#### 2.3.3 전체보기 `[P2]`
1. 전체보기 클릭 시 우선순위 상·중·하에 해당하는 모든 과업을 확인할 수 있다.

### 2.4 과업 진행 바텀시트

#### 2.4.1 기본 상태 `[P1]`
1. 플레이리스트에 하나 이상의 과업이 추가된 경우 홈 화면 하단에 과업 진행 바텀시트를 표시한다. 플레이리스트에 과업이 없는 경우 바텀시트는 표시하지 않는다.
2. 바텀시트에는 플레이리스트의 첫 번째 과업명, 사용자 권장 시간 대비 실제 소요 시간의 진행 정도를 나타내는 진행바, 카테고리 색상 및 아이콘을 표시한다.
3. 실제 소요 시간이 권장 시간과 동일하거나 이를 초과한 경우 진행바는 100% 상태로 유지한다.
4. 플레이리스트에 과업이 처음 추가된 경우 바텀시트에는 과업명과 권장 시간만 표시하며, 진행바는 0% 상태로 초기화된다. 최초 상태는 중지 상태로 표시된다.
5. 사용자가 바텀시트를 클릭하면 해당 과업의 플레이 화면으로 이동한다.
6. 플레이 화면 최초 진입 시 과업은 중지 상태로 표시되며, 사용자가 직접 재생 버튼을 클릭해야 타이머가 시작된다.
7. 바텀시트는 홈 화면, 마이페이지, 캘린더 화면에서만 제공한다.
8. 마이페이지의 하위 화면(내 완료한 과업 보기, 중요한 과업 선택하기, 시간표 설정, 알림 설정)에서는 바텀시트를 표시하지 않는다.

#### 2.4.2 확장 상태 `[P1]`
1. 바텀시트를 위로 올리면 현재 과업의 플레이 화면으로 이동한다.

### 2.5 검색

#### 2.5.1 과업 검색 `[P1]`
1. 상단 검색 버튼 클릭 시 검색 화면으로 이동한다.
2. 검색 화면에서는 기본적으로 최근 검색어를 최대 5개 표시한다.
3. 최근 검색어가 없는 경우 비워놓은 채로 검색창만 둔다.

#### 2.5.2 실시간 추천 `[P1]`
1. 사용자가 검색어를 입력하는 동안 연관 과업을 실시간으로 추천한다.

#### 2.5.3 검색 결과 `[P1]`
1. 검색 완료 시 검색 결과 개수와 연관 과업 목록을 표시한다.

#### 2.5.4 검색 결과 없음 `[P1]`
1. 검색 결과가 없는 경우 "검색 결과가 없습니다." 문구를 표시한다.

### 2.6 과업 조합 추천

#### 2.6.1 추천 모드 `[P2]`
1. 긴급 모드, 불끄기 모드 등 여러 추천 모드를 제공한다.
2. 추천 로직은 logic-spec의 "4. 과업 조합 추천 로직" 참고.

### 2.7 플레이리스트 추가 플로우

#### 2.7.1 과업 카드 클릭 `[P1]`
1. 사용자가 하나의 과업 카드를 클릭하면 플레이리스트에 추가, 수정, 삭제 버튼을 표시한다.
2. 단일 과업만 클릭 가능하다.

#### 2.7.2 "현재 재생 목록에 추가" 클릭 시 — 플레이리스트가 비어있는 경우 `[P1]`
1. 사용자가 플레이리스트에 추가 버튼을 클릭하면 해당 과업을 플레이리스트에 추가한다.
2. 플레이리스트가 비어있는 경우 과업 진행 바텀시트를 표시한다.

#### 2.7.3 "현재 재생 목록에 추가" 클릭 시 — 플레이리스트에 하나 이상 과업이 존재하는 경우 `[P1]`
1. 사용자가 플레이리스트에 추가 버튼을 클릭하면 해당 과업을 현재 플레이리스트에 추가한다.
2. 현재 화면을 유지한 채 "추가되었습니다." 토스트 메시지를 표시한다.

#### 2.7.4 과업 조합 추천의 다운로드 버튼 클릭 시 `[P2]`
1. 과업 조합 추천 영역의 다운로드 버튼 클릭 시 해당 과업 조합을 플레이리스트에 추가한다.
2. 플레이리스트에 과업이 없는 경우 다운로드 버튼 클릭 시 추가된 과업 리스트의 첫 번째 과업의 플레이 화면으로 이동한다.
3. 플레이리스트에 하나 이상의 과업이 존재하는 경우 다운로드 버튼 클릭 시 현재 화면을 유지한 채 "재생 목록에 추가되었습니다." 토스트 메시지를 표시한다.
4. 다운로드 버튼 클릭 후 해당 버튼은 체크 아이콘으로 변경된다.
5. 체크 아이콘 클릭 시 "리스트에 추가된 모든 과업을 삭제하시겠습니까?" 팝업을 표시한다.
6. 사용자가 삭제 버튼을 클릭한 경우 해당 과업 조합(모드)을 통해 플레이리스트에 추가된 모든 과업을 삭제한다.
7. 과업 삭제 완료 시 체크 아이콘은 다시 다운로드 버튼으로 변경된다.

### 2.8 플레이 화면

#### 2.8.1 플레이 화면 구성 `[P1]`
1. 과업명과 우선순위 태그(상·중·하)를 표시한다.
2. 사용자가 기록한 "어디까지 했는지" 메모를 표시한다. 메모가 없는 경우 메모 영역을 표시하지 않는다.
3. 권장 시간과 권장 시간에 대응하는 모자이크 진행바를 표시한다.
4. 모자이크 진행바는 하나당 5분을 가리킨다.
5. 플레이 화면 최초 진입 시 과업 시작 전에는 권장 시간만 표시하며, 과업 시작(재생 버튼 클릭) 시에 소요 시간을 나타낸다.

#### 2.8.2 최초 진입 툴팁 `[P1]`
1. 최초 진입 시 피드백 화면 이동 버튼에 대한 툴팁을 표시한다.
2. 사용자가 화면을 터치하기 전까지 툴팁을 유지한다.

#### 2.8.3 재생 시작 `[P1]`
사용자가 재생 버튼을 클릭하면 공부 시간에 따라 진행바를 채운다.
1. 재생 시작 시 권장 시간 대신 소요 시간을 표시한다.
2. 소요 시간과 진행바는 플레이리스트 전체의 누적 학습 시간을 기준으로 한다.
3. 사용자가 다음 과업으로 이동하더라도 소요 시간과 진행바는 초기화되지 않고 누적 상태를 유지한다.

#### 2.8.4 권장 시간 초과 `[P1]`
1. 권장 시간을 초과한 경우 다른 색상의 진행바로 표시한다.

#### 2.8.5 플레이리스트 바텀시트 `[P1]`
1. 하단 바텀시트를 위로 올리면 플레이리스트에 추가된 과업 목록을 확인할 수 있다.
2. 플레이리스트 전체 삭제 기능 및 일부 삭제 기능을 제공한다.
3. 드래그 핸들을 이용하여 플레이리스트 내 과업 순서를 변경할 수 있다.
4. 상단에는 전체, 불끄기 모드, 밸런스 모드, 해치우기 모드, 찍먹 모드 등의 필터 칩을 제공한다.
5. 사용자가 특정 모드 칩을 선택한 경우 해당 모드에 포함되는 과업만 필터링하여 표시한다.
6. 하나의 플레이리스트에 여러 모드의 과업이 함께 포함될 수 있으므로, 사용자는 필터 칩을 이용하여 원하는 모드의 과업만 확인할 수 있다.
7. 전체 칩 선택 시 사용자가 플레이리스트에 추가한 모든 과업을 표시한다.
8. 선택한 모드에 해당하는 과업이 존재하지 않는 경우 "OO 모드에 해당하는 과업이 없습니다." 문구를 표시한다. (예: "불끄기 모드에 해당하는 과업이 없습니다.")

#### 2.8.6 플레이리스트 내 모든 과업 삭제 시 `[P1]`
1. 사용자가 플레이리스트의 모든 과업을 삭제한 경우 플레이리스트가 비어있는 상태의 플레이 화면을 표시한다.
2. 사용자가 해당 플레이 화면을 아래로 내리면 플레이 화면은 닫히며, 기존 화면이 다시 표시된다.
3. 플레이리스트에 과업이 존재하지 않는 경우 과업 진행 바텀시트는 표시하지 않는다.

### 2.9 피드백 화면

#### 2.9.1 화면 이동 `[P1]`
1. 체크 버튼 클릭 시 피드백 화면으로 이동한다.

#### 2.9.2 메모 기록 `[P1]`
1. 어디까지 수행했는지 기록할 수 있다.
2. 피드백 화면에서는 진척도 입력 없이 메모만 작성도 가능하다.

#### 2.9.3 진척도 입력 — 과업을 세부 단계로 쪼개지 않은 경우 `[P1]`
1. 드래그를 통해 진행한 만큼 진척도를 입력할 수 있다.

#### 2.9.4 진척도 입력 — 과업을 세부 단계로 쪼갠 경우 `[P1]`
1. 단계를 추가한 경우 각 단계별 진척도를 개별적으로 조절할 수 있다.
2. 이때 전체 진척도는 각 단계의 진척도를 반영하여 자동 계산된다. (전체 진행도를 사용자가 직접 조정할 수 없음 — 비활성 상태)
3. 전체 진행도 계산 방식: logic-spec의 "2. 전체 진행률 계산 방식" 참고.
4. 사용자가 단계 쪼개기 버튼을 클릭하면 과업은 기본적으로 3개의 단계로 분할된다.
5. 생성된 단계에는 각각 "개념 정리", "문제 풀이", "전체 복습하기" 플레이스홀더를 기본 제공한다.
6. 사용자는 기본 제공된 플레이스홀더를 자유롭게 수정할 수 있다.

#### 2.9.5 피드백 완료 — 다음 재생할 과업이 존재하는 경우 `[P1]`
1. 피드백 완료 시 다음 과업의 플레이 화면으로 이동하고 "피드백을 완료했습니다." 토스트 메시지를 표시한다.
2. 이동한 다음 과업의 플레이 화면은 중지 상태로 표시되며 사용자가 재생 버튼을 눌러야 시작된다.

#### 2.9.6 피드백 완료 — 다음 재생할 과업이 없는 경우 `[P1]`
1. 피드백 완료 시 현재 과업의 플레이 화면을 중지한 상태에서 "피드백을 완료했습니다." 토스트 메시지를 표시한다.

#### 2.9.7 피드백 화면 하단의 완료 버튼 `[P1]`
1. 완료 버튼은 메모 또는 진척도 둘 중 하나만 입력해도 활성화 상태가 된다. (기본은 비활성 상태)

#### 2.9.8 임시저장 `[P1]`
1. 사용자가 피드백 완료 전에 화면을 이탈한 경우 작성 중이던 메모와 진척도를 임시 저장한다.
2. 다시 피드백 화면에 진입한 경우 임시 저장된 내용을 표시한다.

### 2.10 과업 추가 바텀시트

#### 2.10.1 화면 진입 `[P1]`
1. 홈 화면에서 할 일 추가 버튼 클릭 시 과업 추가 바텀시트를 표시한다.

#### 2.10.2 과업 정보 입력 `[P1]`
1. 사용자는 과업명을 입력할 수 있다.

#### 2.10.3 기본값 설정 `[P1]`
1. 최초 진입 시 사용자가 생성한 첫 번째 카테고리가 기본값으로 선택된다.
2. 1일 전 알림이 기본값으로 선택된다.
3. 반복 안함이 기본값으로 선택된다.
4. 마감일은 미설정 상태로 시작한다.
5. 사용자는 추후 원하는 값으로 변경할 수 있다.

#### 2.10.4 카테고리 설정 `[P1]`
1. 카테고리 태그 클릭 시 카테고리 선택 바텀시트로 이동한다.

#### 2.10.5 알림 설정 `[P1]`
1. 알림 태그 클릭 시 알림 설정 화면으로 이동한다.

#### 2.10.6 반복 설정 `[P1]`
1. 반복 태그 클릭 시 반복 설정 바텀시트로 이동한다.

#### 2.10.7 다음 버튼 `[P1]`
1. 다음 버튼 클릭 시 비슷한 과업 추가 바텀시트로 이동한다.
2. 다음 버튼은 과업명과 마감일자를 반드시 입력해야만 활성화되며, 그 전까지는 비활성 상태이다.

#### 2.10.8 날짜 선택 `[P1]`
1. 날짜 태그 클릭 시 날짜 선택 바텀시트로 이동한다.

### 2.11 카테고리 선택 바텀시트

#### 2.11.1 카테고리 적용 `[P1]`
1. 사용자가 하나의 카테고리를 선택하면 선택 화면은 닫힌다.
2. 선택한 카테고리명이 과업 추가 바텀시트의 태그에 반영된다.

#### 2.11.2 카테고리 편집 `[P1]`
1. 카테고리 편집 버튼 클릭 시 카테고리 편집 바텀시트로 이동한다.

### 2.12 카테고리 편집 바텀시트

#### 2.12.1 카테고리 목록 `[P1]`
1. 사용자가 등록한 카테고리를 버튼 형태로 제공한다.
2. 한 화면에는 3~4개의 카테고리를 표시한다.
3. 기본 카테고리는 학업, 일상, 기념일이다.

#### 2.12.2 기존 카테고리 수정 `[P1]`
1. 기존 카테고리 클릭 시 이름과 색상이 선택된 상태의 화면으로 이동하여 수정할 수 있다.
2. 수정 완료 시 카테고리 편집 바텀시트로 다시 이동한다.
3. 카테고리 추가/수정 화면에서 기존에 사용자가 선택했던 카테고리 색상은 노출하지 않는다.

#### 2.12.3 새 카테고리 추가 `[P1]`
1. 카테고리 추가하기 버튼 클릭 시 이름과 색상이 비어 있는 화면으로 이동하여 새로운 카테고리를 추가할 수 있다.
2. 카테고리 추가 완료 시 카테고리 편집 바텀시트로 다시 이동한다.
3. 카테고리가 하나만 남아있을 때에도 삭제 가능하다.
4. 이름이 같은 카테고리는 생성 가능하다.
5. 기존에 선택했던 색상과 동일한 색상을 가진 카테고리는 생성할 수 없다.
6. 카테고리 추가/수정 화면에서 기존에 사용자가 선택했던 카테고리 색상은 노출하지 않는다.

#### 2.12.4 카테고리 삭제 `[P3]`
1. 삭제 버튼 클릭 시 삭제 여부를 묻는 팝업을 표시한다.
2. 삭제 완료 시 카테고리 편집 바텀시트로 이동한다.

### 2.13 알림 설정 화면

#### 2.13.1 알림 선택 `[P3]`
1. 사용자는 10분 전부터 1주일 전까지의 알림 옵션 중 하나를 선택할 수 있다.
2. 선택 완료 시 알림 선택 바텀시트는 내려가고, 선택된 알림 시간이 반영된 과업 추가 바텀시트로 이동한다.
3. 알림은 단일 선택만 가능하다.

### 2.14 날짜 선택 바텀시트

#### 2.14.1 기본값 `[P1]`
1. 날짜 태그 클릭으로 진입 시 날짜 선택 바텀시트는 기본적으로 '일반' 섹션이 선택되어 있다.

#### 2.14.2 일반 섹션 `[P1]`
1. 사용자가 일반 섹션을 클릭하면 캘린더를 표시한다.
2. 사용자는 캘린더에서 과업의 마감일을 선택할 수 있다.
3. 날짜 선택 시 선택한 마감일이 적용된 과업 등록 바텀시트로 이동한다.

#### 2.14.3 캘린더 `[P1]`
1. 캘린더는 현재 일자가 포함된 월부터 최대 3년 후까지의 월을 스와이프하여 탐색할 수 있다. (왼쪽으로는 스와이프 불가)
2. 사용자는 하루 단위로 날짜를 선택할 수 있다. (단일 선택)
3. 캘린더 상단의 년/월 영역을 클릭하면 년/월 선택 피커를 표시한다.

#### 2.14.4 년/월 선택 피커 `[P1]`
1. 년도와 월은 각각 독립적으로 스크롤하여 선택할 수 있다.
2. 년도는 현재 년도부터 3년 후까지 선택할 수 있다.
3. 현재 년도를 선택한 경우 월은 현재 월부터 12월까지 선택할 수 있다.
4. 다음 년도부터 3년 후까지의 년도를 선택한 경우 월은 1월부터 12월까지 선택할 수 있다.
5. 년 또는 월 선택 시 해당 년/월로 캘린더를 이동한다.

#### 2.14.5 반복 정보 입력 `[P3]`
1. 반복 유형, 시작 날짜, 종료 날짜를 설정할 수 있다.
2. 시작 날짜와 종료 날짜 클릭 시 캘린더 바텀시트를 표시한다.
3. 날짜 선택 시 반복 설정 바텀시트로 돌아간다.

#### 2.14.6 반복 유형 선택 `[P3]`
1. 매주, 매월, 매년 반복을 선택할 수 있다.
2. 반복 설정 완료 시 "반복 설정" 태그가 적용된 과업 등록 바텀시트로 이동한다.
3. 반복 유형은 기본적으로 매주가 선택되어 있다.

#### 2.14.7 매주 반복 `[P3]`
1. 일요일부터 토요일까지 표시하며 다중 선택할 수 있다.

#### 2.14.8 매월 반복 `[P3]`
1. 1일부터 31일까지와 마지막날 버튼을 제공한다.
2. 다중 선택할 수 있다.
3. 마지막날 선택 시 각 월의 마지막 날짜에 반복된다.
4. 31일을 선택했는데 해당 월에 31일이 없는 경우 알림을 제공하지 않는다.

#### 2.14.9 매년 반복 `[P3]`
1. 월은 하나만 선택할 수 있다.
2. 일은 다중 선택할 수 있다.
3. 선택한 월의 선택한 일에 반복된다.
4. 선택한 일이 해당 연도에 존재하지 않는 경우 해당 일자에 대한 알림은 제공하지 않는다.

### 2.16 비슷한 과업 추가 바텀시트

#### 2.16.0 화면 이동 `[P1]`
1. 과업 추가 바텀시트에서 다음 버튼 클릭 시 비슷한 과업 추가 바텀시트로 이동한다.

#### 2.16.1 과업 목록 조회 `[P1]`
1. 기존 과업들을 카테고리별로 분류하여 표시한다.
2. 사용자는 검색을 통해 원하는 과업을 찾을 수 있다.

#### 2.16.2 과업 카드 정보 `[P1]`
1. 각 과업 카드는 카테고리, 과업명, 총 소요 시간, 완료 일자를 표시한다.
2. 총 소요 시간은 시와 분까지만 표시한다.
3. 완료 일자는 그 과업에 등록되었던 마감일자를 기준으로 한다.

#### 2.16.3 버튼 상태 `[P1]`
1. 최초 진입 시: 선택 버튼은 비활성화, "비슷한 경험이 없어요" 버튼은 활성화 상태로 표시된다.
2. 사용자가 과업을 선택했을 때: 선택 버튼은 활성화되고 "비슷한 경험이 없어요" 버튼은 비활성화된다.
3. 선택 해제 시 다시 초기 상태로 돌아간다.

#### 2.16.4 과업 추가 완료 `[P1]`
1. 선택 버튼 또는 "비슷한 경험이 없어요" 버튼 클릭 시 과업 추가를 완료한다.
2. 완료 후 홈 화면으로 이동하며 "과업이 추가되었습니다." 토스트 메시지를 표시한다.

### 2.17 과업 수정 & 삭제 플로우

#### 2.17.1 과업 옵션 표시 `[P1]`
1. 사용자가 하나의 과업 카드를 클릭하면 플레이리스트에 추가, 수정, 삭제 버튼을 표시한다.

#### 2.17.2 과업 삭제 `[P1]`
1. 사용자가 삭제 버튼을 클릭하면 삭제 여부를 묻는 팝업창을 표시한다.
2. 팝업에서 삭제 버튼을 클릭하면 해당 과업을 삭제한다.
3. 삭제 완료 후 홈 화면을 표시하며 "삭제 완료되었습니다." 토스트 메시지를 표시한다.

#### 2.17.3 과업 수정 `[P1]`
1. 사용자가 수정 버튼을 클릭하면 과업 등록 화면으로 이동한다.
2. 과업 등록 화면은 신규 등록 화면과 동일한 구조를 가진다.
3. 기존에 입력했던 과업명, 카테고리, 마감일, 알림, 반복 여부 등의 정보가 기본값으로 입력된 상태로 표시된다.
4. 사용자는 기존 정보를 수정하여 저장할 수 있다.

## 3. 캘린더

### 3.1 기본 화면

#### 3.1.1 화면 진입 `[P1]`
1. 네비게이션바의 '캘린더' 클릭 시 캘린더와 홈 화면에서 등록한 할 일 리스트를 표시한다.
2. 최초 진입 시 오늘 날짜가 기본 선택된다.
3. 홈 화면에서 추가한 과업들을 캘린더에 나타낸다.

#### 3.1.2 캘린더에서 과거 일자 클릭 시 `[P2]`
1. 사용자가 과거 일자를 선택한 경우 해당 일자를 마감일로 갖는 과업 목록을 표시한다.

### 3.2 캘린더 — 일정 개수 표시 `[P1]`
1. 캘린더에는 사용자가 등록한 과업 중 해당 날짜를 마감일로 설정한 과업 수만큼 동그라미(●)를 표시한다.
2. 각 동그라미의 색상은 과업에 설정된 카테고리의 색상을 따른다. 이를 통해 날짜별 과업 수와 카테고리 분포를 한눈에 확인할 수 있다.

### 3.3 통계

#### 3.3.1 통계 화면 진입 `[P1]`
1. 일정 등록 화면 우측 상단의 통계 버튼 클릭 시 통계 화면으로 이동한다.

### 3.4 통계 — 캘린더

#### 3.4.1 일별 시간 표시 `[P1]`
1. 과거 및 오늘 날짜의 경우 사용자가 실제 수행한 시간을 색상과 함께 표시한다.
2. 미래 날짜의 경우 하루 권장 시간을 표시한다.
3. 하루 권장 시간 계산 로직은 logic-spec의 "5. 오늘 최소 시간 로직" 참고.

### 3.5 통계 — 캘린더 하단

#### 3.5.1 과업 진행 기록 `[P1]`
1. 사용자가 피드백을 완료한 과업만 리스트로 표시한다.
2. 사용자가 드래그한 진척도만큼 진행 상태를 반영한 과업 카드를 표시한다.
3. 100% 완료한 과업의 경우 체크 표시로 나타낸다.

### 3.6 통계 — 기록 없음 `[P1]`
1. 오늘 날짜이지만 피드백을 완료하지 않은 경우 "아직 기록이 없어요" 문구를 표시한다.
2. 미래 날짜를 선택한 경우 "아직 기록이 없어요" 문구를 표시한다.

## 4. 마이페이지

### 4.1 기본 화면

#### 4.1.1 메뉴 구성 `[P1]`
1. 마이페이지 진입 시 내 완료한 과업 보기, 중요한 과업 선택하기, 시간표 설정, 알림 설정, 로그아웃 메뉴를 제공한다.

### 4.2 로그아웃

#### 4.2.1 로그아웃 확인 `[P1]`
1. 로그아웃 버튼 클릭 시 로그아웃 여부를 묻는 팝업창을 표시한다.
2. 사용자가 로그아웃을 선택하면 로그아웃된 온보딩 화면으로 이동한다.

### 4.3 내 완료한 과업 보기

#### 4.3.1 완료 과업 조회 `[P1]`
1. 사용자가 완료한 과업 목록을 표시한다.
2. 각 과업 카드는 카테고리명, 과업명, 총 소요 시간, 완료 일자를 표시한다.
3. 완료 일자는 마감일자를 기준으로 표시한다.
4. 최대 3년 전까지의 완료한 과업을 볼 수 있다.

#### 4.3.2 검색 `[P1]`
1. 사용자가 과업명을 검색할 수 있다.
2. 검색 중에는 실시간으로 연관 과업을 표시한다.
3. 검색 완료 시 검색 결과 개수와 함께 검색 결과를 표시한다.
4. 검색 결과가 없는 경우 "검색 결과가 없습니다." 문구를 표시한다.

### 4.4 중요한 과업 선택하기

#### 4.4.1 과업 조회 `[P1]`
1. 현재 진행 중인 과업만 표시한다.
2. 마감일이 지난 과업은 표시하지 않는다.
3. 우선순위 상·중·하에 따라 섹션을 구분하여 표시한다.

#### 4.4.2 우선순위 설정 `[P1]`
1. 모든 과업의 별표 버튼은 기본적으로 비활성화 상태로 표시한다.
2. 사용자가 별표 버튼을 클릭하면 활성화 상태로 변경된다.
3. 변경된 우선순위는 실시간으로 반영된다.

#### 4.4.3 검색 `[P1]`
1. 사용자가 과업명을 검색할 수 있다.
2. 검색 중에는 실시간으로 연관 과업을 표시한다.
3. 검색 완료 시 검색 결과 개수와 함께 검색 결과를 표시한다.
4. 검색 결과가 없는 경우 "검색 결과가 없습니다." 문구를 표시한다.

### 4.5 시간표 등록

#### 4.5.1 기본 화면 `[P1]`
1. 시간표의 기본 시간 범위는 08:00~24:00이다.
2. 추가 버튼과 설정 버튼을 제공한다.

#### 4.5.2 시간 선택 `[P3]`
1. 사용자는 웬투밋(when2meet) 형태로 원하는 시간대를 드래그하여 한 번에 여러 시간대를 선택할 수 있다.
2. 30분 단위로 드래그 가능하다.
3. 사용자는 30분 단위로 시간을 삭제할 수 있다.

#### 4.5.3 시간 삭제 및 확장 `[P3]`
1. 기존 선택 영역 내부를 다시 드래그하면 해당 구간만 삭제된다.
2. 기존 선택 영역 내부에서 시작하여 바깥까지 드래그하면 드래그 시작 시점부터 기존 시간대가 삭제된다. (추가 없이 삭제만 이루어짐)
3. 기존 선택 영역보다 앞에서 시작하여 겹치도록 드래그하면 기존 시간대와 합쳐져 확장된다.
4. 터치한 구간은 30분 단위로 선택 및 해제가 가능하다.

#### 4.5.4 완료 버튼 `[P3]`
1. 사용자가 시간대를 선택한 후 완료 버튼을 클릭하면 일정 추가 바텀시트를 표시한다.

#### 4.5.5 일정 등록 바텀시트 `[P3]`
1. 사용자가 시간대를 드래그한 뒤 완료 버튼을 클릭하면 일정 등록 바텀시트를 표시한다.
2. 사용자는 이름, 장소, 색상을 선택할 수 있다.
3. 시간은 사용자가 드래그한 모든 시간대를 표시한다.
4. 시간대가 많을 경우 바텀시트를 스크롤하여 전체 시간대를 확인할 수 있다.
5. 시간 추가 버튼 클릭 시 타임휠 바텀시트를 표시한다.

#### 4.5.7 타임휠 바텀시트 `[P1]`
1. 타임휠 바텀시트에서는 요일과 시작 시간, 종료 시간을 선택할 수 있다.
2. 확인 버튼 클릭 시 선택한 시간이 시간 추가 바텀시트에 반영된다.
3. 취소 버튼 클릭 시 초기화된 시간 추가 바텀시트로 이동한다.

#### 4.5.8 시간 추가 완료 `[P3]`
1. 타임휠 바텀시트에서 확인 버튼 클릭 시 일정 등록 바텀시트로 이동한다.
2. 일정 등록 바텀시트에는 사용자가 드래그한 시간과 추가한 시간이 모두 표시된다.
3. 기존 등록된 시간과 일부라도 겹치는 경우 "시간표가 겹칩니다." 팝업을 표시한다.

#### 4.5.9 시간 수정 플로우 `[P3]`
1. 일정 등록 바텀시트에서 특정 시간 클릭 시 그 시간이 적용된 타임휠 바텀시트를 표시한다.
2. 일정 등록 바텀시트에서 특정 요일 클릭 시 그 요일이 적용된 타임휠 바텀시트를 표시한다.

### 4.6 시간표 설정

#### 4.6.1 시간표 설정 `[P2]`
1. 시작 시간과 종료 시간을 수정할 수 있다.
2. 기본값은 시작 시간 08:00, 종료 시간 24:00이다.

#### 4.6.2 초기화 `[P3]`
1. 초기화 버튼 클릭 시 삭제 여부를 묻는 팝업을 표시한다.
2. 삭제 버튼 클릭 시 등록된 모든 시간표를 삭제한 화면으로 이동한다.

#### 4.6.3 에브리타임 연동 `[P3]`
1. 사용자는 에브리타임 시간표 링크를 붙여넣어 연동할 수 있다.
2. 확인 버튼은 기본적으로 비활성화 상태이며 링크 입력 시 활성화된다.

#### 4.6.4 에브리타임 연동 결과 `[P3]`
1. 연동 성공 시 추가된 시간표 화면으로 이동한다.
2. 연동 실패 시 비어있는 시간표 화면을 표시한다.
3. "등록한 시간표가 없습니다." 토스트 메시지를 표시한다.

### 4.7 시간표 수정

#### 4.7.1 시간표 수정 `[P3]`
1. 사용자가 등록된 하나의 시간표를 클릭하면 바텀시트를 표시한다.
2. 바텀시트에는 기존에 입력한 일정명, 장소, 시간 정보가 선택된 상태로 표시된다.
3. 시간은 등록된 시간표의 모든 시간대를 표시한다.
4. 수정 버튼 클릭 시 해당 정보들이 입력된 일정 등록 바텀시트를 띄워 일정명, 장소, 색상, 시간을 수정할 수 있다.

#### 4.7.3 시간표 삭제 `[P3]`
1. 사용자가 삭제 버튼을 클릭하면 선택한 시간표를 삭제한다.
2. 삭제 시 사용자가 선택한 시간대의 시간표만 삭제한다.
3. 동일한 일정명 또는 과업명을 사용하는 다른 시간표는 함께 삭제되지 않는다.
4. 삭제 후 수정 바텀시트를 닫고 변경된 시간표를 표시한다.

### 4.8 알림 설정

#### 4.8.1 기본 화면 `[P3]`
1. 서비스 알림과 30분 Pack 알림 항목을 제공한다.
2. 각 항목별 On/Off 토글 버튼을 제공한다.
3. 최초 진입 시 모든 알림은 OFF 상태로 표시한다.

#### 4.8.2 서비스 알림 `[P3]`
1. 사용자는 서비스 알림 수신 여부를 설정할 수 있다.
2. 토글 활성화 시 서비스 관련 알림을 수신한다.
3. 토글 비활성화 시 서비스 관련 알림을 수신하지 않는다.
4. 과업 등록 완료 시 기기의 알림 권한 요청 팝업을 표시한다.
5. 사용자가 권한을 허용한 경우 알림을 수신할 수 있다.
6. 사용자가 권한을 거부한 경우 알림을 수신할 수 없다.

#### 4.8.3 30분 Pack 알림 `[P3]`
1. 사용자는 30분 Pack 알림 수신 여부를 설정할 수 있다.
2. 토글 활성화 시 30분 Pack 알림을 수신한다.
3. 토글 비활성화 시 30분 Pack 알림을 수신하지 않는다.
4. 시간표 설정 시 기기의 30분 Pack 알림 권한 요청 팝업을 표시한다.
5. 사용자가 권한을 허용한 경우 알림을 수신할 수 있다.
6. 사용자가 권한을 거부한 경우 알림을 수신할 수 없다.

#### 4.8.4 권한 미허용 `[P3]`
1. 기기의 알림 권한이 비활성화된 경우 ON으로 알림 설정 변경 시 권한 활성화 안내 문구를 표시한다.
2. 사용자는 기기 설정 화면으로 이동하여 알림 권한을 변경할 수 있다.

## 5. 권한 허용

### 5.1 알림 권한 허용 `[P3]`

**(1) 기본 알림 권한**
1. 사용자가 최초로 홈 화면에 진입할 경우 시스템은 기본 알림 권한 허용 팝업을 노출한다.
2. 사용자가 권한을 허용할 경우 마이페이지의 기본 알림 설정은 ON 상태로 변경된다.
3. 사용자가 권한을 거부하거나 허용하지 않을 경우 마이페이지의 기본 알림 설정은 OFF 상태로 유지된다.
4. 사용자가 마이페이지에서 OFF 상태의 알림 토글을 ON으로 변경하려고 할 경우 시스템은 다시 알림 권한 허용 팝업을 노출한다.
5. 사용자가 권한을 허용하면 알림 설정은 ON으로 변경되며, 허용하지 않을 경우 OFF 상태를 유지한다.

**(2) 30분 Pack 알림 권한**
1. 사용자가 시간표 등록을 완료할 경우 시스템은 30분 Pack 알림 권한 허용 팝업을 노출한다.
2. 사용자가 권한을 허용할 경우 마이페이지의 30분 Pack 알림 설정은 ON 상태로 변경된다.
3. 사용자가 권한을 거부하거나 허용하지 않을 경우 마이페이지의 30분 Pack 알림 설정은 OFF 상태로 유지된다.
4. 사용자가 마이페이지에서 OFF 상태의 30분 Pack 알림 토글을 ON으로 변경하려고 할 경우 시스템은 다시 30분 Pack 알림 권한 허용 팝업을 노출한다.
5. 사용자가 권한을 허용하면 알림 설정은 ON으로 변경되며, 허용하지 않을 경우 OFF 상태를 유지한다.
